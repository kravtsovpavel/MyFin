# MyFin — Личный финансовый трекер

Веб-приложение для учёта личных финансов с поддержкой тёмной темы, аналитикой и синхронизацией через Supabase.

## 🚀 Быстрый старт

### 1. Настройка Supabase

1. Создайте проект на [supabase.com](https://supabase.com)
2. Включите Google OAuth в Authentication → Providers
3. Выполните SQL-скрипт для настройки БД:
   - Откройте SQL Editor в панели Supabase
   - Скопируйте содержимое файла `supabase-setup.sql`
   - Нажмите "Run"

### 2. Настройка Google OAuth

1. Создайте проект в [Google Cloud Console](https://console.cloud.google.com)
2. Создайте OAuth 2.0 credentials
3. Добавьте redirect URI: `https://YOUR_PROJECT_ID.supabase.co/auth/v1/callback`
4. В панели Supabase укажите:
   - Google Client ID
   - Google Client Secret

### 3. Конфигурация приложения

Откройте файл `js/config.js` и замените значения:

```javascript
const SUPABASE_CONFIG = {
    url: 'https://YOUR_PROJECT_ID.supabase.co',
    anonKey: 'YOUR_ANON_KEY'
};
```

Получить ключи можно в Settings → API в панели Supabase.

### 4. Локальный запуск

Просто откройте `index.html` в браузере или используйте локальный сервер:

```bash
# Используем Python
python -m http.server 8000

# Или Node.js (npx)
npx serve .

# Или VS Code Live Server
```

Перейдите по адресу `http://localhost:8000`

### 5. Деплой на GitHub Pages

1. Создайте репозиторий на GitHub
2. Запушьте файлы проекта
3. В настройках репозитория:
   - Settings → Pages
   - Source: Deploy from a branch
   - Branch: main (или master)
   - Folder: / (root)
4. Через 1-2 минуты сайт будет доступен по адресу:
   `https://YOUR_USERNAME.github.io/YOUR_REPO/`

## 📱 Функционал

### Авторизация
- Вход через Google OAuth
- Сессия до 30 дней
- Автоматический вход при повторном открытии

### Дашборд
- Доходы, расходы, баланс за месяц
- Динамика к прошлому месяцу (▲/▼)
- График доходов/расходов по дням
- Прогресс бюджета
- Последние 10 операций

### Добавление транзакций
- Выбор типа (доход/расход) — **нет значения по умолчанию**
- Сумма, категория, дата, комментарий
- Редактирование и удаление

### Категории
- **Нет категорий по умолчанию**
- Пользователь создаёт свои категории
- До 30 категорий для доходов и расходов
- Эмодзи для каждой категории

### Аналитика
- Линейный график динамики
- Круговые диаграммы по категориям
- Сравнение с прошлым периодом
- Переключение периодов (день/неделя/месяц/год)

### Календарь трат
- Цветовая индикация дней с расходами
- Детали по клику на день
- Переключение месяцев

### Все операции
- Пагинация (25 операций на страницу)
- Фильтры по типу, категории, периоду
- Экспорт в Excel

### Настройки
- Выбор валюты (₽, $, €, ₸, Br)
- Первый день недели (пн/вс)
- Месячный бюджет на расходы
- Экспорт всех данных в Excel
- Удаление всех данных

## 🔔 Уведомления

- При достижении 90% бюджета — предупреждение
- При превышении 100% — уведомление о превышении

## 📊 База данных

**Таблицы:**
- `profiles` — профили пользователей
- `categories` — категории доходов/расходов
- `transactions` — транзакции

**RLS (Row Level Security):**
- Пользователи видят только свои данные
- Все операции защищены политиками безопасности

## 🛠 Технологии

- HTML5, CSS3, Vanilla JavaScript (ES6+)
- Chart.js — графики
- Supabase — база данных + авторизация
- SheetJS (XLSX) — экспорт в Excel

## 📝 Заметки

```javascript
const SUPABASE_CONFIG = {
    url: 'https://your-production-project.supabase.co',

};
```

### Обновление приложения

При изменении кода на GitHub Pages:
1. Запушьте изменения в репозиторий
2. GitHub Pages автоматически обновится через 1-2 минуты
3. При необходимости очистите кэш браузера (Ctrl+Shift+R)
